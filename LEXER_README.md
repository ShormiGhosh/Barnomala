`# বর্ণমালা (Barnomala) — Lexer Explanation (`blexer.l`)

This document explains **every line** of the `blexer.l` file — the lexical analyzer (lexer/scanner) for the Bangla programming language.

A lexer's job is to read raw source code characters and convert them into **tokens** — meaningful units like keywords, numbers, operators, identifiers.

---

## File Structure Overview

A Flex (`.l`) file has **3 sections** separated by `%%`:

```
%{ ... %}       ← Section 1: C declarations (copied directly to output)
%%              ← Section 2: Pattern-action rules (the actual lexer rules)
%%              ← Section 3: User code (helper functions)
```

---

## Section 1: C Declarations (`%{ ... %}`)

```c
%{
```
**Purpose:** Opens the C declaration block. Everything between `%{` and `%}` is copied verbatim into the generated C file (`lex.yy.c`).

---

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
```
**Purpose:** Standard C library headers.
- `stdio.h` — for `printf`, `fprintf`, `FILE`, `EOF`
- `stdlib.h` — for `malloc`, `free`, `atoi`, `atof`
- `string.h` — for `strdup`, `strlen`

---

```c
#include "bparser.tab.h"
```
**Purpose:** Includes the header generated by Bison (the parser). This header defines all token constants (`IF`, `ELSE`, `PRINT`, `INT_LITERAL`, etc.) as integer values. Without this, the lexer wouldn't know what `return IF;` means.

---

### Indentation Tracking Variables

```c
#define MAX_INDENT_DEPTH 100
```
**Purpose:** Maximum nesting depth for indentation. Supports up to 100 levels of nested blocks. Like Python, our language uses indentation to define code blocks.

```c
int indent_stack[MAX_INDENT_DEPTH];
```
**Purpose:** A **stack** (array used as stack) that stores the indentation level (number of spaces) at each nesting depth. For example:
- `indent_stack[0] = 0` → top-level code (no indentation)
- `indent_stack[1] = 4` → first level of indentation (4 spaces)
- `indent_stack[2] = 8` → second level (8 spaces)

```c
int indent_top = 0;
```
**Purpose:** Stack pointer — index of the top element in `indent_stack`. Starts at 0 (top-level).

```c
int pending_dedents = 0;
```
**Purpose:** Counter for how many DEDENT tokens still need to be emitted. When indentation decreases by multiple levels at once (e.g., from 8 spaces back to 0), we need to emit multiple DEDENTs. Flex can only return one token per rule match, so we store the extras here.

```c
int pending_indent = 0;
```
**Purpose:** Flag (0 or 1) indicating that an INDENT token needs to be emitted before the next real token. Set to 1 when the newline handler detects increased indentation.

```c
int paren_depth = 0;
```
**Purpose:** Tracks how many open parentheses/brackets are unmatched. When `paren_depth > 0`, we're inside `(...)` or `[...]`, so newlines should be ignored (no NEWLINE/INDENT/DEDENT tokens). This allows function calls and array initializations to span multiple lines.

---

### Indentation Helper Functions

```c
void push_indent(int level) {
    if (indent_top < MAX_INDENT_DEPTH - 1) {
        indent_stack[++indent_top] = level;
    }
}
```
**Purpose:** Push a new indentation level onto the stack. Called when indentation increases (entering a new block). The `++indent_top` increments the pointer first, then stores the value.

**Example:** If current indent is 0 and we see 4 spaces → `push_indent(4)` → stack becomes `[0, 4]`, `indent_top = 1`.

---

```c
int pop_indent() {
    if (indent_top > 0) {
        return indent_stack[indent_top--];
    }
    return 0;
}
```
**Purpose:** Pop (remove and return) the top indentation level from the stack. Called when indentation decreases (exiting a block). The `indent_top--` returns the value first, then decrements.

---

```c
int get_current_indent() {
    return indent_top > 0 ? indent_stack[indent_top] : 0;
}
```
**Purpose:** Returns the current indentation level (top of stack) without removing it. Returns 0 if stack is at base level.

---

```c
void init_indent() {
    indent_top = 0;
    indent_stack[0] = 0;
    pending_dedents = 0;
    pending_indent = 0;
    paren_depth = 0;
}
```
**Purpose:** Resets all indentation state to initial values. Called from `main()` before parsing begins.

---

### The YY_DECL Trick

```c
#define YY_DECL int internal_yylex(void)
```
**Purpose:** This is the **key trick** for our indentation system. Normally, Flex generates a function called `yylex()` that the parser calls. By redefining `YY_DECL`, we tell Flex to name its generated function `internal_yylex()` instead.

Then we write our own `yylex()` wrapper (at the bottom of the file) that:
1. First checks if there are pending INDENT/DEDENT tokens to emit
2. If yes, returns those without calling the real scanner
3. If no, calls `internal_yylex()` to get the next token from the file

**Why?** Flex can only return ONE token per pattern match. But when indentation changes, we sometimes need to emit NEWLINE + INDENT, or NEWLINE + multiple DEDENTs. The wrapper solves this.

```c
%}
```
**Purpose:** Closes the C declaration block.

---

## Flex Options and Start Conditions

```c
%x COMMENT
```
**Purpose:** Declares `COMMENT` as an **exclusive start condition**. When the lexer enters COMMENT state (via `BEGIN(COMMENT)`), ONLY rules prefixed with `<COMMENT>` are active. All other rules are disabled. This is how we handle multi-line comments — everything between `মন্তব্য` and `মন্তব্য_শেষ` is ignored.

- `%x` = exclusive (only matching rules active)
- `%s` = inclusive (matching rules AND rules without prefix are active)

---

```c
%option noyywrap
```
**Purpose:** Tells Flex we don't need the `yywrap()` function. Normally, when the lexer reaches end-of-file, it calls `yywrap()` to check if there are more files to process. Since we only process one file, we don't need it.

---

```c
%option yylineno
```
**Purpose:** Tells Flex to automatically track the current line number in the variable `yylineno`. This is used for error messages like "Syntax error at line 18".

---

## Section 2: Pattern-Action Rules (`%%`)

```c
%%
```
**Purpose:** Marks the beginning of the rules section. Each rule is: `pattern { action }`. Flex tries to match patterns against the input text, and when it finds a match, it executes the corresponding action.

**Important Flex matching rules:**
1. **Longest match wins** — if multiple patterns match, the longest match is used
2. **First rule wins** — if two patterns match the same length, the first one in the file wins
3. `yytext` — contains the matched text
4. `yyleng` — contains the length of the matched text

---

### Newline Handler (The Heart of Indentation)

```c
\n {
```
**Purpose:** Matches a newline character (`\n`). This is the most complex rule because it handles the entire indentation detection logic.

```c
    if (paren_depth > 0) {
        /* ignore newlines inside parens */
    }
```
**Purpose:** If we're inside parentheses or brackets (e.g., `নাও(ইউজার_নাম)` or `[০, ১, ২]`), ignore the newline completely. Don't emit NEWLINE, INDENT, or DEDENT. This allows multi-line expressions.

```c
    } else {
        int c;
        int indent;
```
**Purpose:** Variables for the lookahead loop:
- `c` — holds each character we read ahead
- `indent` — counts spaces at the start of the next line

```c
        while (1) {
            indent = 0;
            while ((c = input()) == ' ') {
                indent++;
            }
```
**Purpose:** Inner loop reads characters one by one using `input()` (a Flex built-in function that reads the next character from the input stream). Counts how many spaces are at the beginning of the next line.

```c
            if (c == '\t') {
                indent += 4;
                continue;
            }
```
**Purpose:** If a tab character is encountered, treat it as 4 spaces and continue counting.

```c
            if (c == '\r') continue;
```
**Purpose:** Skip carriage return characters (Windows line endings `\r\n`).

```c
            if (c == '\n') continue;
```
**Purpose:** If the next line is blank (just another newline), skip it and try the next line. This is why blank lines don't cause problems in the source code.

```c
            break;
        }
```
**Purpose:** When we find a non-whitespace, non-newline character, we've found the start of actual code on the next line. Break out of the loop.

```c
        if (c != EOF && c != 0) {
            unput(c);
        }
```
**Purpose:** Put back the non-space character we just read, so the lexer can process it normally in the next call. `unput()` is a Flex built-in that pushes a character back onto the input stream.

```c
        int current = get_current_indent();
```
**Purpose:** Get the indentation level of the current block.

```c
        if (indent > current) {
            push_indent(indent);
            pending_indent = 1;
        }
```
**Purpose:** If the next line is **more indented** than the current level, we're entering a new block. Push the new level onto the stack and set the flag to emit an INDENT token before the next real token.

**Example:**
```
যদি বয়স বড় ১৮ঃ     ← indent=0
    দেখাও "প্রাপ্তবয়স্ক"।  ← indent=4, which is > 0, so INDENT
```

```c
        } else if (indent < current) {
            while (indent_top > 0 && get_current_indent() > indent) {
                pop_indent();
                pending_dedents++;
            }
        }
```
**Purpose:** If the next line is **less indented**, we're exiting one or more blocks. Pop levels off the stack until we match the new indentation level, counting how many DEDENTs to emit.

**Example:**
```
        যোগফল = যোগফল যোগ গ।  ← indent=8
    ফিরিয়ে_দাও যোগফল।         ← indent=4, pop once → 1 DEDENT
দেখাও উত্তর।                  ← indent=0, pop twice → 2 DEDENTs
```

```c
        return NEWLINE;
    }
}
```
**Purpose:** Return the NEWLINE token to the parser. The wrapper function `yylex()` will emit any pending INDENT/DEDENT tokens BEFORE returning this NEWLINE (or after, depending on the sequence).

### Simple Character Rules

```c
\r { /* Skip carriage return */ }
```
**Purpose:** Silently ignore `\r` characters. Handles Windows line endings (`\r\n`) — the `\r` is skipped, and `\n` is handled by the newline rule above.

```c
[ \t]+ { /* Skip whitespace within lines */ }
```
**Purpose:** Skip spaces and tabs that appear in the middle of a line (between tokens). The `+` means one or more. This does NOT match indentation at line start — that's handled by the newline rule's lookahead.

### Punctuation and Delimiters

```c
"("                 { paren_depth++; return LPAREN; }
")"                 { paren_depth--; return RPAREN; }
```
**Purpose:** Match parentheses. `paren_depth++` tracks that we're inside parens (so newlines are ignored). Returns `LPAREN`/`RPAREN` tokens.

```c
"["                 { paren_depth++; return LBRACKET; }
"]"                 { paren_depth--; return RBRACKET; }
```
**Purpose:** Same as above but for square brackets. Used in array declarations like `ধ[৫]`.

```c
","                 { return COMMA; }
```
**Purpose:** Match comma — used in function parameters and array elements.

```c
"ঃ"                 { return COLON; }
```
**Purpose:** Match the Bengali visarga character `ঃ` (U+0983) used as a colon after `যদি`, `নাহলে`, `যতক্ষণ`, etc. to start a block.

```c
"।"                 { return SEMICOLON; }
```
**Purpose:** Match the Bengali danda `।` (U+0964) used as a statement terminator (like `;` in C).

### Program Structure Keywords

**Purpose:** Match the program start and end markers. Every Bangla program must begin with `শুরু_প্রোগ্রাম` and end with `শেষ_প্রোগ্রাম`.

### Include Statement


### String and Character Literals

```c
\"([^\"]|\\\")*\"   { yylval.strval = strdup(yytext); return STRING_LITERAL; }
```
**Purpose:** Match a double-quoted string. The regex breakdown:
- `\"` — opening double quote
- `([^\"]|\\\")*` — zero or more of: any character except `"`, OR an escaped quote `\"`
- `\"` — closing double quote
- `strdup(yytext)` — copies the entire matched string (including quotes) as the token's value

**Example:** `"আমি ভাত খাই"` matches as a single STRING_LITERAL.

```c
\'(\xE0[\xA6\xA7][\x80-\xBF]|[^'\n\\]|\\.)\'  { yylval.strval = strdup(yytext); return CHAR_LITERAL; }
```
**Purpose:** Match a single-quoted character literal. Handles:
- Bengali characters: `\xE0[\xA6\xA7][\x80-\xBF]` matches UTF-8 Bengali chars (3 bytes starting with E0 A6 or E0 A7)
- ASCII characters: `[^'\n\\]` — any char except quote, newline, backslash
- Escape sequences: `\\.` — backslash followed by any character

**Example:** `'ক'` matches as a CHAR_LITERAL.

### Bangla Number Literals

#### Float (Decimal) Numbers

```c
(\xE0\xA7[\xA6-\xAF])+"."(\xE0\xA7[\xA6-\xAF])+ {
```
**Purpose:** Match a Bangla floating-point number like `৩.১৪`. The pattern:
- `(\xE0\xA7[\xA6-\xAF])+` — one or more Bangla digits (০-৯ are E0 A7 A6 through E0 A7 AF)
- `"."` — a literal dot
- `(\xE0\xA7[\xA6-\xAF])+` — one or more Bangla digits after the dot

```c
    char *english = strdup(yytext);
    int len = strlen(yytext);
    int pos = 0;
    for(int i = 0; i < len; ) {
        if(i + 2 < len && (unsigned char)english[i] == 0xE0 &&
           (unsigned char)english[i+1] == 0xA7 &&
           (unsigned char)english[i+2] >= 0xA6 &&
           (unsigned char)english[i+2] <= 0xAF) {
            int digit = (unsigned char)english[i+2] - 0xA6;
            english[pos++] = '0' + digit;
            i += 3;
        } else {
            english[pos++] = english[i++];
        }
    }
    english[pos] = '\0';
    yylval.floatval = atof(english);
    free(english);
    return FLOAT_LITERAL;
}
```
**Purpose:** Converts Bangla digits to English digits, then parses as a C float.

**Conversion logic:**
- Each Bangla digit is 3 bytes: E0 A7 (A6+digit)
- `০` = E0 A7 **A6** → A6 - A6 = 0 → `'0'`
- `১` = E0 A7 **A7** → A7 - A6 = 1 → `'1'`
- `৯` = E0 A7 **AF** → AF - A6 = 9 → `'9'`
- The dot `.` passes through as-is

**Example:** `৩.১৪` → "3.14" → `atof("3.14")` → 3.14

#### Integer Numbers

```c
(\xE0\xA7[\xA6-\xAF])+ {
```
**Purpose:** Same pattern but WITHOUT the dot — matches integer Bangla numbers like `১৮`, `৫`, `১০০`. Same conversion logic, but uses `atoi()` instead of `atof()` and returns `INT_LITERAL`.

**Note:** This rule appears AFTER the float rule. Flex's longest-match ensures `৩.১৪` matches the float rule (longer), not `৩` as an integer.

### Comments

```c
"মন্তব্য"              { BEGIN(COMMENT); }
```
**Purpose:** When the lexer sees `মন্তব্য` (comment), it switches to the COMMENT exclusive state. In this state, only `<COMMENT>` rules are active.

```c
<COMMENT>{
    "মন্তব্য_শেষ"           { BEGIN(INITIAL); }
    .|\n                 { }
}
```
**Purpose:**
- `"মন্তব্য_শেষ"` — when we see the comment-end marker, switch back to INITIAL (normal) state via `BEGIN(INITIAL)`
- `.|\n` — any character (`.`) or newline (`\n`) inside a comment is silently consumed (empty action `{ }`)

**Example:**
```
মন্তব্য এটি একটি মন্তব্য মন্তব্য_শেষ
```
Everything between `মন্তব্য` and `মন্তব্য_শেষ` is ignored.

### Identifier Pattern

```c
(\xE0\xA6[\x80-\x82\x84-\xBF]|\xE0\xA7[\x80-\xBF]|_)(\xE0\xA6[\x80-\x82\x84-\xBF]|\xE0\xA7[\x80-\xBF]|\xE0\xA7[\xA6-\xAF]|_)* { yylval.strval = strdup(yytext); return IDENTIFIER; }
```
**Purpose:** Match a Bangla identifier (variable name, function name). This is the most complex pattern.

**First character** (must be a letter or `_`):
- `\xE0\xA6[\x80-\x82\x84-\xBF]` — Bengali letters in U+0980-U+0982, U+0984-U+09BF range
- `\xE0\xA7[\x80-\xBF]` — Bengali letters/signs in U+09C0-U+09FF range
- `_` — underscore (for multi-word names like `সত্য_মিথ্যা`)

**Note:** `\x83` is excluded from the first range — this is `ঃ` (visarga), used as our colon character, so it shouldn't start an identifier.

**Subsequent characters** (same as above, plus digits):
- Same letter ranges as first character
- `\xE0\xA7[\xA6-\xAF]` — Bangla digits ০-৯ (allowed in identifiers but not as first char)
- `_` — underscore

**Why this rule is LAST (before the catch-all):** Flex's longest-match rule ensures keywords like `যদি` match their specific rules first. If the input has `যদিও`, it's longer than `যদি`, so the IDENTIFIER rule wins — correct behavior!

### Catch-All Rule

```c
.                    { fprintf(stderr, "Unknown character: %s at line %d\n", yytext, yylineno); }
```
**Purpose:** Matches any single character that wasn't matched by any rule above. Prints an error message with the unknown character and line number. The `.` pattern matches any character except `\n`.

### End-of-File Handler

```c
<<EOF>> {
    if (pending_dedents > 0) {
        pending_dedents--;
        return DEDENT;
    }
    if (indent_top > 0) {
        pop_indent();
        return DEDENT;
    }
    yyterminate();
}
```
**Purpose:** When the lexer reaches end of file, it must emit DEDENT tokens for any still-open blocks. 

**Example:** If the file ends while inside an indented block:
```
যদি condition ঃ
    দেখাও "হ্যাঁ"।
শেষ_প্রোগ্রাম          ← EOF after this
```
The DEDENT for the `যদি` block must be emitted before the parser sees EOF. Without this, the parser would complain about a missing DEDENT.

- First drains `pending_dedents` (one at a time, since each call returns one token)
- Then checks `indent_top > 0` for any remaining open blocks
- Finally, `yyterminate()` signals end-of-input to the parser

### The yylex Wrapper Function

```c
int yylex(void) {
    if (pending_indent) {
        pending_indent = 0;
        return INDENT;
    }
    if (pending_dedents > 0) {
        pending_dedents--;
        return DEDENT;
    }
    return internal_yylex();
}
```
**Purpose:** This is the function the **parser actually calls**. It wraps the Flex-generated `internal_yylex()`:

1. **Check `pending_indent`:** If the newline handler detected increased indentation, emit an INDENT token first (before the next real token on the indented line).

2. **Check `pending_dedents`:** If there are pending DEDENT tokens (indentation decreased), emit one DEDENT per call until all are drained.

3. **Call `internal_yylex()`:** If no pending tokens, call the real Flex scanner to read the next token from the input file.

**Token sequence example** for:
```
যদি x বড় ৫ঃ
    দেখাও x।
দেখাও y।
```

| Call # | Returns | Why |
|--------|---------|-----|
| 1 | IF | `যদি` matched |
| 2 | IDENTIFIER | `x` matched |
| 3 | GREATER | `বড়` matched |
| 4 | INT_LITERAL | `৫` matched |
| 5 | COLON | `ঃ` matched |
| 6 | NEWLINE | `\n` matched, lookahead sees 4 spaces → sets pending_indent=1 |
| 7 | INDENT | wrapper emits pending INDENT |
| 8 | PRINT | `দেখাও` matched |
| 9 | IDENTIFIER | `x` matched |
| 10 | SEMICOLON | `।` matched |
| 11 | NEWLINE | `\n` matched, lookahead sees 0 spaces → sets pending_dedents=1 |
| 12 | DEDENT | wrapper emits pending DEDENT |
| 13 | PRINT | `দেখাও` matched |
| ... | ... | ... |

## Summary

The lexer converts Bangla source code into a stream of tokens:
- **Keywords** → specific token types (IF, ELSE, WHILE, etc.)
- **Identifiers** → IDENTIFIER with the name as value
- **Numbers** → INT_LITERAL or FLOAT_LITERAL with converted value
- **Operators** → PLUS, MINUS, GREATER, etc.
- **Punctuation** → LPAREN, SEMICOLON, COLON, etc.
- **Indentation** → INDENT, DEDENT, NEWLINE (virtual tokens for block structure)
- **Comments** → silently ignored
- **Whitespace** → silently ignored (except indentation)
